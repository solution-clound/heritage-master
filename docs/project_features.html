1. 非遗项目搜索与详情
5 项 ▼
1.1 关键词/类别/地区搜索
按关键词、十大类别、省份地区筛选非遗项目。爬取 ihchina.cn 官方 API，内置 30+ 条兜底数据。
heritage_search.py crawler.py GET /api/search search_heritage_projects
1.2 项目详情查询
多源降级：ihchina.cn 详情页 → knowledge.json → 百度百科 → 内置数据 → 元数据生成描述。
heritage_search.py crawler.py GET /api/project/{name} get_heritage_project_info
1.3 非遗十大类别
返回非遗十大类别列表（民间文学、传统音乐、传统舞蹈、传统戏剧、曲艺、传统美术、传统技艺、民俗等）。
GET /api/categories get_heritage_categories
1.4 LLM 增强搜索
搜索结果可选 LLM 摘要增强。组合端点同时返回项目 + 活动 + 场馆 + 路线建议。
GET /api/search?enrich=True GET /api/search/enriched
1.5 百度百科集成
从百度百科抓取非遗项目补充信息。
crawler.py GET /api/baike/{keyword}

2. 场馆查找（高德地图集成）
4 项 ▼
2.1 城市场馆搜索
调高德 POI API，支持文本搜索和坐标附近搜索，返回名称、地址、评分、电话等。
venue_finder.py GET /api/venues find_heritage_venues
2.2 场馆详情
查询单个场馆的详细信息：地址、电话、营业时间、类型、区域、照片。
GET /api/venue/detail
2.3 地图配置
返回高德 JS API Key，供前端地图渲染使用。
GET /api/map-config
2.4 AmapView 地图组件
前端交互式地图组件，支持标记点、路线绘制。
AmapView.vue

Haversine 距离计算 + 高德步行导航 + 最近邻 TSP 启发式 + 按天聚类。返回文字行程和结构化路线数据（供前端地图渲染）。
route_planner.py POST /api/trip plan_heritage_trip

10 个主要城市预置路线，当无实时场馆数据时使用。
route_planner.py

knowledge_base.py

6. AI 探索助手（Agent）
5 项 ▼
6.1 对话式 Agent + Function Calling
自动路由到搜索/场馆/旅行三个工具，最多 5 轮工具调用循环。
agent_tools.py POST /api/agent
6.2 个性化注入 (NEW)
用户画像 + 记忆 + 兴趣标签 + 阶段行为 + 个性化指令注入 system prompt。根据 question_count 调整回复深度。
app.py: _build_agent_personalization()
6.3 个性化问候 (NEW)
记忆感知的欢迎语，引用近期话题和核心记忆，返回用户兴趣标签供前端动态生成快捷标签。
GET /api/agent/greeting
6.4 服务端对话历史 (NEW)
对话历史存服务端数据库，支持跨设备同步。登录用户从服务端加载，未登录用户保持 localStorage。
GET /api/agent/conversations GET /api/agent/conversations/{id}
6.5 前端个性化欢迎 (NEW)
根据用户兴趣动态生成快捷标签（替代固定标签），显示个性化问候文案。
AgentView.vue api.js

7. 大师人格系统
5 项 ▼
7.1 内置大师人格
2 个大师：叶汉钟（潮州工夫茶）、陈少峰（广东醒狮）。每个大师包含 system_prompt、场景描述、三阶段行为、问候语、专业领域、性格特征。
master_prompt.py

7.3 三阶段自适应提示
试探期→信任期→修行期。每阶段控制：开放度、留白比例、问候风格、教学深度、主动教学。
master_prompt.py
7.4 MCP 提示模板
4 个 MCP 提示：专家提示、对比分析、导览、深度探索。
heritage_expert_prompt heritage_compare heritage_guide heritage_deep_dive
7.5 人格蒸馏脚本
从语录/采访中用 LLM 自动提取人格特征，生成 OpenPersona JSON。
scripts/distill_persona.py

10. 长期记忆系统
4 项 ▼
10.1 记忆管理器（文件 + Redis 缓存）
读穿透：Redis → JSON 文件 → 创建空白。写穿透：文件 → Redis。核心记忆 CRUD，6 种记忆类型（interest/milestone/preference/personality/event/feedback）。
memory.py: MemoryManager
10.2 记忆提取（LLM + 规则降级）
LLM 从对话中提取记忆，失败时降级为关键词规则匹配。支持画像推断（每 5 个问题推断兴趣/性格更新）。
memory.py: MemoryExtractor
10.3 个性化问候生成
根据缺席天数、核心记忆、教学进度、关系阶段生成个性化开场白。
memory.py: GreetingGenerator
10.4 记忆整理压缩
记忆超阈值（默认 50 条）时，LLM 自动合并、去重、压缩。
memory.py: consolidate_memories()
